## 一、产品概述

### 1.1 产品定位

一款专为个体服装商户设计的安卓离线进销存管理应用。支持商品管理、销售记账、进货入库、账单查询、退货退款、库存查看、热销排行、利润趋势等功能。所有数据存储在设备本地，无需网络。

### 1.2 目标用户

**种子用户**：我妈，48岁，服装摊主，非技术背景。

**经营环境**：服装市场摊位，环境嘈杂，网络不稳定。

**核心痛点**：
- 手工纸笔记账易丢失
- 库存靠翻货架
- 晚上收摊后需花大量时间对账

### 1.3 核心价值

- **10秒完成记账**：点击商品 → 确认售价 → 完成销售
- **离线可用**：无需网络，所有数据本地存储
- **灵活扩展**：分类、属性完全由用户自定义
- **隐私保护**：账单数据默认隐藏，查询后才显示


## 二、用户场景

| 场景 | 描述 |
| :--- | :--- |
| 快速销售 | 顾客买一件红色M码连衣裙，摊主点分类→点商品→确认售价→记账成功 |
| 砍价处理 | 顾客砍价，销售时直接修改售价 |

## 四、交互设计

### 4.1 核心设计原则

- **拇指友好**：所有可点击元素高度至少48dp，主按钮56dp
- **三步完成**：核心记账操作不超过三步
- **点选代替输入**：分类、属性通过下拉框或标签选择，减少文字输入
- **出错可逆**：所有删除、退款等敏感操作均需二次确认，并伴有音效提示
- **默认不暴露数据**：账单需主动查询才显示金额；首页不显示收入数字
- **视觉一致**：所有页面采用绿色渐变背景，半透明白色卡片，大圆角按钮，胶囊形底部导航

### 4.2 视觉风格

- **主题色**：稳重绿（#43A047）
- **状态栏**：沉浸式绿色（与标题栏融为一体）
- **页面背景**：绿色渐变（#C8E6C9 → #A5D6A7 → #F1F8E9）
- **标题栏**：绿色背景 + 白色文字（56dp高）
- **大按钮**：28dp大圆角、绿色实心白字（56dp高）
- **中等按钮**：28dp圆角、绿色实心白字（44dp高）
- **卡片**：16dp圆角、半透明白色背景、轻阴影
- **底部导航**：白色胶囊容器、选中绿底白字、未选中透明底灰字
- **对话框按钮**：绿色实心白字、圆角

### 4.3 导航体验

- **底部导航固定**：所有主页面（首页、账单、进货、我的）底部导航栏固定在屏幕底部，不随内容滚动
- **统一切换动画**：点击导航按钮时使用淡入淡出动画（`fade_in` / `fade_out`），页面切换平滑自然
- **当前页面标识**：被选中的导航按钮显示为绿色实心背景 + 白色文字，其他按钮为透明背景 + 灰色文字

### 4.4 状态反馈

- **操作成功**：Toast提示 + 对应音效（卖出、入库、退款等各有不同音效）
- **操作失败**：Toast提示 + 错误音效
- **二次确认**：弹出对话框 + 确认提示音
- **删除操作**：弹出确认对话框，确认后播放删除音效
- **取消操作**：播放取消音效


## 五、技术方案

| 层级 | 技术选型 |
| :--- | :--- |
| 平台 | Android 原生（Java） |
| 最低SDK | Android 11（API 30） |
| UI框架 | XML布局 + 自定义Drawable样式 |
| 数据库 | Room（SQLite） |
| 架构模式 | 多Activity + BaseActivity基类（统一音效、过渡动画、异常处理、导航方法） |
| 音效实现 | MediaPlayer + res/raw中的MP3文件 |
| 网络依赖 | 零（完全离线） |


## 六、数据库设计

| 表名 | 说明 |
| :--- | :--- |
| `products` | 商品基础信息（含图片路径、进价、售价） |
| `categories` | 商品分类 |
| `attribute_groups` | 属性组（风格/面料/颜色等） |
| `attribute_values` | 属性值 |
| `product_attributes` | 商品与属性关联（多对多） |
| `sales` | 销售记录（含退款标记） |
| `purchases` | 进货记录 |

> 库存通过 `累计进货 - 累计销售（不含退款）` 实时计算。
