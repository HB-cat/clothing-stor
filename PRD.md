## 一、产品概述

### 1.1 产品定位

一款专为个体服装商户设计的安卓离线进销存管理应用。支持商品管理、销售记账、进货入库、账单查询、退货退款、库存查看、热销排行、利润趋势等功能。所有数据存储在设备本地，无需网络。

### 1.2 目标用户

**种子用户**：我妈，48岁，服装摊主，非技术背景。

**经营环境**：服装市场摊位，环境嘈杂，网络不稳定。

**核心痛点**：
- 手工纸笔记账易丢失
- 库存靠翻货架
- 晚上收摊后需花大量时间对账

### 1.3 核心价值

- **10秒完成记账**：点击商品 → 确认售价 → 完成销售
- **离线可用**：无需网络，所有数据本地存储
- **灵活扩展**：分类、属性完全由用户自定义
- **隐私保护**：账单数据默认隐藏，查询后才显示


## 二、用户场景

| 场景 | 描述 |
| :--- | :--- |
| 快速销售 | 顾客买一件红色M码连衣裙，摊主点分类→点商品→确认售价→记账成功 |
| 砍价处理 | 顾客砍价，销售时直接修改售价 |
| 到货入库 | 新款到货，选择商品，填写数量进价，一键入库 |
| 顾客退货 | 账单页面左滑记录，二次确认后退款 |
| 晚上看账 | 收摊后打开账单，选日期查询，查看当日销售额、毛利 |
| 新增品类 | 来了新品类，去分类管理添加，首页自动出现 |
| 扩展属性 | 想按"季节"管理，去属性管理添加，录商品时自动出现 |

## 四、交互设计

### 4.1 核心设计原则

- **拇指友好**：所有按钮足够大，单手可操作
- **三步完成**：核心记账操作不超过三步
- **点选代替输入**：能用选择的绝不让用户打字
- **出错可逆**：敏感操作均有二次确认
- **默认不暴露数据**：账单需主动查询才显示金额

### 4.2 视觉风格

- **主题色**：稳重绿（#43A047）
- **背景**：绿色渐变（#C8E6C9 → #A5D6A7 → #F1F8E9）
- **按钮**：28dp大圆角，绿色实心白字
- **卡片**：16dp圆角，半透明白色背景
- **导航**：圆角胶囊，选中绿底白字
- **字体**：清晰易读，标题18-22sp，正文14-16sp

## 六、数据库设计

| 表名 | 说明 |
| :--- | :--- |
| `products` | 商品基础信息 |
| `categories` | 商品分类 |
| `attribute_groups` | 属性组（风格/面料/颜色等） |
| `attribute_values` | 属性值 |
| `product_attributes` | 商品与属性关联（多对多） |
| `sales` | 销售记录（含退款标记） |
| `purchases` | 进货记录 |

> 库存通过 `累计进货 - 累计销售（不含退款）` 实时计算。
