================================================================================
  一、产品概述
================================================================================

  1.1 产品定位
      一款专为个体服装商户设计的安卓离线进销存管理应用。V1.4及之前版本为
      单用户离线版本，所有数据存储在设备本地SQLite数据库，无需网络。

  1.2 目标用户
      个体服装摊主（以我妈为种子用户），非技术背景。
      经营环境: 服装市场摊位，环境嘈杂，网络不稳定。

  1.3 核心价值
      - 10秒完成记账: 点击商品 -> 确认售价 -> 完成销售
      - 离线可用: 纯本地运行，无需网络
      - 灵活扩展: 分类、属性完全由用户自定义
      - 隐私保护: 账单数据默认隐藏，查询后才显示
      - 经营洞察: 规则引擎自动分析数据，生成经营建议


================================================================================
  二、用户场景
================================================================================

  场景        描述
  ----------  ----------------------------------------------------------------
  快速销售    顾客买一件红色M码连衣裙，摊主点分类->点商品->确认售价->记账成功
  砍价处理    顾客砍价，销售时直接修改售价
  到货入库    新款到货，选择商品，填写数量进价，一键入库
  晚上看账    收摊后打开账单，选日期查询，查看当日销售额、毛利
  经营分析    打开经营分析页，看今日快报和智能建议，了解什么好卖、该补什么
  新增品类    来了新品类，去分类管理添加，首页自动出现
  扩展属性    想按"季节"管理，去属性管理添加，录商品时自动出现


================================================================================
  三、功能需求
================================================================================

  3.1 商品管理
      功能          描述                                      优先级
      ------------  ----------------------------------------  ------
      添加商品      名称、分类、属性标签、进价、售价、拍照    P0
      商品列表      按分类筛选展示，列表项显示图片、进价、售价 P0

  3.2 分类管理
      功能          描述                        优先级
      ------------  --------------------------  ------
      动态分类      用户可新增/删除分类          P0
      预置分类      连衣裙类/T恤类/衬衫类等默认  P0
      二次确认      删除分类时弹出确认框          P0

  3.3 属性管理
      功能          描述                                    优先级
      ------------  --------------------------------------  ------
      属性组管理    风格/面料/颜色/尺码/季节等，用户可新增   P0
      属性值管理    批量添加、单独删除，均有二次确认          P0
      预置属性      风格、面料、颜色、尺码默认存在            P0
      智能提示      添加属性值时根据组名自动生成提示文字      P1

  3.4 销售记账
      功能          描述                              优先级
      ------------  --------------------------------  ------
      点击记账      选分类->选商品->确认售价->记账     P0
      快捷指令      输入"卖了连衣裙一件90块"自动解析   P0
      改价支持      销售时可临时修改售价               P0

  3.5 进货入库
      功能          描述                            优先级
      ------------  ------------------------------  ------
      批量入库      多属性行，填写数量和进价          P0
      自动计算      实时显示总进货金额，自动填充进价  P0
      二次确认      确认入库前弹出确认框              P1

  3.6 账单查询
      功能          描述                                  优先级
      ------------  ------------------------------------  ------
      日/月/年统计  日历选择器 + 滚轮                       P0
      默认隐藏      查询后才显示金额                         P0
      销售流水      列表展示每笔销售详情                     P0
      左滑退款      销售流水左滑弹出退款确认框               P0
      详情查看      点击销售记录弹出详情对话框               P1

  3.7 退货退款
      功能          描述                                  优先级
      ------------  ------------------------------------  ------
      退款执行      确认退款后标记销售记录为已退款            P0
      二次确认      退款操作需两次确认（左滑->弹窗->确认）   P0

  3.8 经营分析 (V1.4新增)
      功能          描述                                    优先级
      ------------  --------------------------------------  ------
      今日快报      今日销售额、毛利、订单数、客单价四卡片    P1
      趋势图        近7天销售趋势折线图，横屏查看，数值标注   P1
      热销分析      识别销量冠军，展示Top 3，横屏柱状图      P1
      滞销提醒      近30天无销量商品，弹窗列表                P1
      补货建议      库存<=5件，弹窗列表，颜色区分紧急度      P1
      时段分析      24小时订单柱状图，横屏查看，数值标注      P1
      规则引擎      BusinessAdvisor类，SQL查询+规则生成建议  P1
      智能体预留    预留IAIProvider接口，未来可切换AI         P2

  3.9 库存查看
      功能          描述                          优先级
      ------------  ----------------------------  ------
      实时计算      进货-销售=当前库存             P0
      低库存预警    库存<=5件红色标识              P0

  3.10 系统功能
      功能          描述                              优先级
      ------------  --------------------------------  ------
      店铺名称        修改后首页标题联动                 P1
      声音反馈        9种操作音效+全局点击音效+可开关    P1
      数据导出        系统文件选择器选择保存位置          P1
      数据恢复        系统文件选择器选择备份文件，0831   P1
      自动备份        每天首次启动自动备份，保留7天      P1
      初始化系统    恢复出厂状态，需授权码0831         P1
      使用指南        内置完整操作说明页面               P1
      可拖拽菜单    "我的"页面功能入口拖拽排序         P1
      异常捕获        全局崩溃日志记录与自动重启          P1


================================================================================
  四、交互设计
================================================================================

  4.1 核心设计原则
      - 拇指友好: 可点击元素 >= 48dp，主按钮 56dp
      - 三步完成: 核心记账操作不超过三步
      - 点选代替输入: 分类、属性通过下拉框或标签选择
      - 出错可逆: 敏感操作均需二次确认
      - 默认不暴露数据: 账单需主动查询才显示金额
      - 结论先行: 经营分析用大号文字给出精简结论

  4.2 视觉风格
      - 主题色: 稳重绿 (#43A047)
      - 状态栏: 沉浸式绿色
      - 页面背景: 绿色渐变 (#C8E6C9 -> #A5D6A7 -> #F1F8E9)
      - 标题栏: 深绿色背景 + 白色文字 (56dp高)
      - 大按钮: 28dp大圆角、绿色实心白字 (56dp高)
      - 卡片: 16dp圆角、半透明白色背景、轻阴影
      - 底部导航: 白色胶囊容器、选中绿底白字、未选中透明底灰字

  4.3 导航体验
      - 底部导航固定: 所有主页面底部导航栏固定在屏幕底部
      - 统一切换动画: 点击导航按钮时使用淡入淡出动画
      - 当前页面标识: 选中按钮绿色实心+白字，其他透明底+灰字

  4.4 经营分析交互 (V1.4新增)
      - 主页分层: 今日快报 -> 各模块精简结论 + 按钮
      - 单入口设计: 每个模块一个"查看"按钮，靠右对齐，半透明圆角
      - 图表横屏: 趋势/热销/时段点击进入横屏全屏图表，含数值标注
      - 列表弹窗: 滞销/补货点击弹出对话框列表

================================================================================
  六、数据库设计
================================================================================

  V1.4 七张核心表:

  表名               说明
  ---------------    ------------------------------
  products           		商品基础信息 (图片路径、进价、售价)
  categories        	 	商品分类
  attribute_groups   	属性组 (风格/面料/颜色等)
  attribute_values   	属性值
  product_attributes 	商品与属性关联 (多对多)
  sales              		销售记录 (含退款标记)
  purchases          		进货记录

  库存通过 "累计进货 - 累计销售（不含退款）" 实时计算，保证数据一致性。
